# 배포 가이드 (Vercel + 기기 간 동기화)

이 앱은 **비밀 암호 하나**로 여러 기기(폰·PC)가 같은 대본 보관함을 공유합니다.
영상 원본까지 클라우드(Vercel Blob)에 저장되어, 폰에서 추출한 결과를 PC에서도 그대로 재생·열람할 수 있습니다.

Gemini 분석 정확도는 기존과 **100% 동일**합니다 (같은 Gemini를 그대로 사용).

---

## 1. GitHub에 코드 올리기

이 앱은 GitHub 저장소 `26.07.04-script-server-version` 의 **최상위**에 올라갑니다
(앱 파일이 repo 루트에 바로 위치).

## 2. Vercel에 프로젝트 연결

1. [vercel.com](https://vercel.com) → **Add New → Project** → `26.07.04-script-server-version` 저장소 선택
2. **Root Directory**: 기본값(루트) 그대로 둡니다 (앱이 이미 최상위에 있음)
3. Framework Preset: **Vite** (자동 감지됨)
4. 일단 **Deploy** 클릭 (아래 3~4번 설정 후 재배포하면 완성)

## 3. Blob 저장소(영상·대본 창고) 만들기

1. Vercel 프로젝트 → **Storage** 탭 → **Create Database → Blob** 선택
2. 이름 아무거나 정하고 생성 → 이 프로젝트에 **Connect**
3. 연결하면 `BLOB_READ_WRITE_TOKEN` 환경변수가 **자동으로** 추가됩니다 (직접 입력 안 해도 됨)

## 4. 비밀 암호 등록 (무단 사용 방지)

1. Vercel 프로젝트 → **Settings → Environment Variables**
2. 새 변수 추가:
   - **Name**: `APP_PASSPHRASE`
   - **Value**: 원하는 비밀 암호 (예: `mysecret2026`) — 남이 모를 값으로
   - 적용 대상: Production/Preview/Development 모두 체크
3. 저장

> 이 암호를 아는 사람/기기만 보관함에 접근할 수 있습니다.
> 나중에 바꾸는 방법은 아래 **"🔑 비밀 암호 변경하기"** 섹션 참고.

## 5. 재배포

환경변수를 넣었으니 **Deployments → 최신 배포 → Redeploy** 로 다시 배포합니다.

## 🔑 비밀 암호 변경하기

> **구조 이해(중요)**: 앱에서 입력하는 암호 = Vercel의 `APP_PASSPHRASE` 값입니다.
> 그리고 이 암호를 SHA-256 해시해서 **클라우드 저장 폴더(네임스페이스)** 를 만듭니다.
> 즉 **암호를 바꾸면 = 다른(빈) 폴더를 가리킴** → 옛 암호로 올린 클라우드 데이터는
> 새 암호에선 안 보입니다(삭제는 아니고 옛 폴더에 그대로 남음).
> 그래서 데이터를 유지하려면 **옛 폴더 → 새 폴더로 복사(이전)** 가 필요합니다.
>
> ⚠️ **앱 안엔 암호 변경 버튼이 없습니다.** 브라우저는 서버 환경변수를 못 바꾸기 때문입니다.
> 변경은 아래처럼 Vercel에서 합니다.

### A. 데이터까지 새 암호로 옮기며 변경 (권장)

**순서 엄수 — Vercel에서 암호 바꾸기 "전에" 먼저 이전!** (이전은 현재=옛 암호로 인증하므로)

1. **이전 실행** (아직 옛 암호인 상태에서) — 브라우저 주소창에:
   ```
   https://<앱주소>/api/migrate-vault?passphrase=옛암호&newPassphrase=새암호
   ```
   - `<앱주소>` 예: `26-07-04-script-server-version.vercel.app`
   - 결과 `{"ok":true,"copied":N,"failed":0}` → N개 복사 완료. `failed`가 0이 아니면 다시 실행(덮어쓰기라 안전).
2. **Vercel에서 암호 변경 + 재배포**
   - Settings → Environment Variables → `APP_PASSPHRASE` → **Edit** → 값에 **새 암호** 입력 → **Save**
   - (참고: Value 칸에 회색 `sk_live_a12…` 같은 글자는 Vercel의 예시 placeholder일 뿐, 실제 값 아님)
   - Deployments → 최신 배포 → **⋯ → Redeploy** (⚠️ 재배포 안 하면 반영 안 됨)
3. **각 기기에서 새 암호로 로그인**
   - 이미 열려 있으면: 설정(⚙️) → **보관함 잠그기** → 새 암호 입력
   - 안 열려 있으면: 암호창에 새 암호 입력
   - → 옛 클라우드 데이터(클라우드에만 있던 것 포함)까지 새 암호에서 그대로 보임

### B. 그냥 암호만 변경 (클라우드 데이터 이전 안 함)

- 위 **2 → 3** 만 실행 (1번 이전 생략)
- 옛 클라우드 데이터는 새 암호에선 안 보임(옛 폴더에 남음). 단, **각 기기 로컬 대본·영상은 그대로**이며,
  온라인일 때 자동 재업로드로 **각 기기가 가진 것 기준으로 새 폴더에 다시 채워짐**.

### 데이터별 영향 요약

| 데이터 | 암호 변경 후 |
|--------|-------------|
| 각 기기 로컬 대본·영상 (localStorage/IndexedDB) | ✅ 그대로 (암호 무관) |
| 클라우드 공유분 — A로 이전함 | ✅ 새 암호에서 그대로 |
| 클라우드 공유분 — B(이전 안 함) | ⚠️ 새 암호선 안 보임(옛 폴더에 보존). 로컬 있는 건 자동 재업로드됨 |

### 도구/참고

- **`/api/migrate-vault`** 는 상시 유지되는 이전 도구입니다. **현재(옛) 암호로 인증**해야만 동작하므로
  아무나 못 씁니다. 재실행해도 덮어쓰기라 안전(멱등). 옛 폴더는 지우지 않고 **복사**만 합니다(문제 시 옛 암호로 복귀 가능).
- 옛 폴더를 나중에 정리(삭제)하고 싶으면 별도 요청. (급하지 않으면 그냥 둬도 무해)

## 사용 방법

1. 배포된 주소(`https://내프로젝트.vercel.app`)에 접속
2. **비밀 암호 입력** (4번에서 정한 `APP_PASSPHRASE` 값)
3. 설정(⚙️)에서 **Gemini API 키 입력** — 보안상 키는 기기마다 각자 입력합니다 (동기화 안 함)
4. 동영상 넣고 추출 → 자동으로 클라우드에 저장됨
5. 다른 기기(폰/PC)에서 같은 주소 접속 → 같은 비밀 암호 입력 →
   목록 화면 하단 **"다른 기기 (클라우드)"** 섹션에서 그 대본을 눌러 열람·재생

## 참고 / 한계

- **무료 용량**: 영상 저장 5GB, 월 데이터 전송 100GB (Vercel Hobby 기준). 저용량 영상이면 수십~백 개 저장 가능.
- **영상 1개는 512MB 이하** 권장 (그 이상은 전송 비용 증가).
- **API 키**는 기기별 입력 (동기화하면 암호 유출 시 결제 키까지 노출되므로 일부러 분리).
- 로컬 저장(localStorage/IndexedDB)도 그대로 유지되어, 인터넷이 없어도 그 기기서 뽑은 건 볼 수 있습니다. 클라우드는 그 위에 얹힌 공유 계층입니다.
